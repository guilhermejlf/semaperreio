### Task 2.2 - Create BudgetEditModal.vue

**Files modified:** `frontend/src/components/BudgetEditModal.vue` (new)

**Action:**

1. Create modal component per UI-SPEC:
   - Props: `visible` (boolean), `meta` (object with categoria, valor_meta, gasto_realizado, etc.)
   - Emits: `@save`, `@cancel`

2. Two-step flow:
   - **Step 1 (Form):** Input for `valor_meta`, pre-filled with current value if editing
     - Show context: "Você já gastou R$ {gasto} nesta categoria"
     - Cancel / Save buttons
   - **Step 2 (Confirmation):** Only shown if editing existing budget with gasto > 0
     - Message: "Você já gastou R$ {gasto} de R$ {old_meta}. Alterar para R$ {new_meta}?"
     - Back / Confirm buttons

3. Validation:
   - `valor_meta` > 0
   - Numeric input with 2 decimal places
   - Visual feedback on invalid

4. Modal animations: fade-in backdrop, scale card (per UI-SPEC)

**Acceptance criteria:**
- [ ] Modal opens/closes with animation
- [ ] Form pre-filled when editing, empty when creating
- [ ] Confirmation step only shows for edits with existing spend
- [ ] Input validation prevents save with invalid value
- [ ] Back button returns to form from confirmation
- [ ] ESC closes modal, returns focus to trigger

### Task 4.2 - Toast Alert on Gasto Add

**Files modified:** `frontend/src/App.vue`, `frontend/src/components/BudgetView.vue` (or toast service)

**Action:**

1. When gasto is added (in App.vue gasto creation flow), check response for `alerta_meta`:
   ```javascript
   if (response.alerta_meta) {
     this.showToast({
       severity: response.alerta_meta.status === 'critical' ? 'error' : 'warn',
       summary: 'Meta de Gastos',
       detail: response.alerta_meta.mensagem,
       life: 5000
     })
   }
   ```

2. Use PrimeVue Toast component (or custom toast if not using PrimeVue Toast)
   - If PrimeVue Toast not installed: create simple custom toast component or use existing notification pattern
   - Position: top-right or bottom-right
   - Auto-dismiss after 5 seconds
   - Color-coded by severity (red for critical, yellow for warning)

3. Alternative if PrimeVue Toast unavailable: add toast state to App.vue, render inline toast component

**Acceptance criteria:**
- [ ] Toast appears when gasto exceeds budget threshold (80% or 100%)
- [ ] Toast message includes category and percentage
- [ ] Toast auto-dismisses after 5 seconds
- [ ] Multiple toasts can queue (not overwrite)
- [ ] No toast if no budget exists or threshold not crossed
