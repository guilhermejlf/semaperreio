## Wave 1: Database Model and Migration

### Task 1.1 - Create Budget/Meta Model

**Files modified:** `api/models.py`, migrations

**Action:**

1. Add `Meta` model to `api/models.py`:
   - `user = ForeignKey(User, on_delete=CASCADE)` - owner of the budget
   - `categoria = CharField(max_length=50, choices=CATEGORIAS_CHOICES, null=True, blank=True)`
     - null = meta geral (total budget for the month)
     - non-null = category-specific budget
   - `mes = PositiveSmallIntegerField(validators=[MinValueValidator(1), MaxValueValidator(12)])`
   - `ano = PositiveSmallIntegerField()`
   - `valor_meta = DecimalField(max_digits=10, decimal_places=2, validators=[MinValueValidator(0.01)])`
   - `created_at = DateTimeField(auto_now_add=True)`
   - `updated_at = DateTimeField(auto_now=True)`
   - `unique_together = [user, categoria, mes, ano]` - one budget per user/category/month/year

2. Create and run migration:
   ```bash
   python manage.py makemigrations api
   python manage.py migrate
   ```

**Acceptance criteria:**
- [ ] `Meta` model exists with all fields
- [ ] Unique constraint prevents duplicate budgets for same user/category/period
- [ ] Migration applies without errors
- [ ] `python manage.py check` passes

## Wave 2: API Endpoints

### Task 2.1 - Budget CRUD Endpoints

**Files modified:** `api/views.py` (or new `api/views_budget.py`), `api/urls.py`, `api/serializers.py`

**Action:**

1. Create `BudgetSerializer` in `api/serializers.py`:
   - Fields: `id`, `categoria`, `categoria_nome` (display), `mes`, `ano`, `valor_meta`, `gasto_realizado` (computed), `percentual_usado` (computed), `status` (computed: 'ok', 'warning', 'danger', 'critical')
   - `gasto_realizado` = sum of user's gastos in that category/month/year (or total if categoria is null)
   - `percentual_usado` = (gasto_realizado / valor_meta) * 100
   - `status` mapping: <50%='ok', 50-80%='warning', >80%='danger', >100%='critical'

2. Add views in `api/views.py` (or `api/views_budget.py`):
   - `GET /api/metas/?mes=&ano=` - list all budgets for user in period
     - Includes computed fields (gasto_realizado, percentual, status)
     - Returns both category budgets + general budget (if exists)
     - If no budgets exist, returns empty list (not 404)
   - `POST /api/metas/` - create budget
     - Body: `categoria` (optional, null for general), `mes`, `ano`, `valor_meta`
     - Validates: user, unique constraint, mes 1-12, ano 4 digits
     - Returns 400 if duplicate budget exists
   - `PUT /api/metas/<id>/` - update budget value
     - Body: `valor_meta`
     - Validates: budget belongs to current user
   - `DELETE /api/metas/<id>/` - delete budget
     - Validates: budget belongs to current user

3. Register URLs in `api/urls.py`:
   ```python
   path('metas/', views.listar_metas),
   path('metas/criar/', views.criar_meta),
   path('metas/<int:pk>/', views.atualizar_meta),
   path('metas/<int:pk>/deletar/', views.deletar_meta),
   ```
   Or use a ModelViewSet with router.

**Acceptance criteria:**
- [ ] `GET /api/metas/?mes=6&ano=2026` returns list with computed fields
- [ ] `POST /api/metas/` creates budget with validation
- [ ] Duplicate budget returns 400 with clear message
- [ ] `PUT` updates valor_meta, preserves other fields
- [ ] `DELETE` removes budget, returns 204
- [ ] All endpoints require authentication (IsAuthenticated)
- [ ] User can only see/modify their own budgets

### Task 2.2 - Dashboard Integration

**Action:**

1. Update `dashboard()` view to include budget data:
   - After computing `ranking_categorias`, add budget progress for each category:
     ```python

       "geral": {"meta": float, "gasto": float, "pct": float, "status": str} or null,

         {"categoria": str, "nome": str, "meta": float, "gasto": float, "pct": float, "status": str}
       ]
     }
     ```
   - Query `Meta` model for user's budgets matching the requested mes/ano
   - For each category in ranking_categorias, check if budget exists
   - For general budget (categoria=null), compute total_gastos vs meta_geral
   - If no budgets exist for period, `metas.geral = null` and `metas.por_categoria = []`

   - If general budget exists and > 80%: `"⚠️ Geral: {pct}% da meta mensal atingida"`

**Acceptance criteria:**
- [ ] Dashboard response includes `metas` object
- [ ] `metas.geral` is null when no general budget set
- [ ] `metas.por_categoria` includes all categories with budgets
- [ ] Computed `pct` is correct: (gasto / meta) * 100

## Wave 3: Check Alert Endpoint

### Task 3.1 - Alert on Gasto Creation

**Files modified:** `api/views.py`

**Action:**

1. Add helper function `check_budget_alert(user, gasto)`:
   - After gasto is saved, check if user's budgets for gasto's month/year/category are exceeded
   - Returns: `{"alertar": bool, "mensagem": str, "categoria": str, "pct": float, "status": str}`
   - Only triggers alert when crossing 80% or 100% threshold (not on every gasto)
   - General budget alert takes precedence over category alert if both apply

2. Modify `gastos` create/update views to call this helper after save
   - Return alert info in response: `{"gasto": {...}, "alerta_meta": {...} or null}`

**Acceptance criteria:**
- [ ] Alert triggers when gasto pushes category over 80% or 100% of budget
- [ ] Alert message includes category name, percentage, and amount over
- [ ] Alert only triggers once per threshold crossing (not repeatedly)
- [ ] Response includes `alerta_meta` field
- [ ] No alert if no budget exists for category/period
