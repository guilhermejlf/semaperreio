## Tasks

### Task 1: Create Family and FamilyMembership models
**File:** `backend/api/models.py`
- Add `Family` model: `name` (CharField 100), `code` (CharField 6, unique), `code_expires_at` (DateTimeField), `created_by` (FK User), `created_at` (DateTimeField auto_now_add)
- Add `FamilyMembership` model: `user` (OneToOneField User, on_delete CASCADE), `family` (FK Family, related_name='members'), `role` (CharField choices admin/member, default='member'), `joined_at` (DateTimeField auto_now_add)
- Add `gasto_created_at` to `Gasto` model (DateTimeField auto_now_add) if not present, for "post-join" edit permission logic
- Migrate: `python manage.py makemigrations api` and `python manage.py migrate`

### Task 2: Create serializers for Family
**File:** `backend/api/serializers.py` (or new `serializers_family.py`)
- `FamilySerializer`: fields `id`, `name`, `code`, `code_expires_at`, `created_by` (read_only), `created_at`
- `FamilyMembershipSerializer`: fields `id`, `user` (nested: id, username, first_name), `role`, `joined_at`
- `FamilyDetailSerializer`: extends FamilySerializer, adds `members` (list of FamilyMembershipSerializer), `member_count`
- `JoinFamilySerializer`: only field `code` (CharField 6, required)
- `CreateFamilySerializer`: only field `name` (CharField 100, required)
- `RegenerateCodeSerializer`: empty (action serializer)

### Task 3: Create Family API views
**File:** `backend/api/views_family.py`
- `FamilyViewSet` (ModelViewSet) with `@action` methods:
  - `GET /api/family/` â†’ `retrieve()` â€” return current user's family (404 if none)
  - `POST /api/family/` â†’ `create()` â€” create family, auto-generate 6-char uppercase code, set expires_at=now+7days, set creator as admin, auto-create membership
  - `DELETE /api/family/` â†’ `destroy()` â€” delete family (admin only), all memberships cascade
  - `POST /api/family/join/` â†’ `@action detail=False, methods=['post']` â€” join via code, validate code not expired, create membership as member, reject if user already in family
  - `POST /api/family/leave/` â†’ `@action detail=False, methods=['post']` â€” leave family; if last admin, reject or force transfer (see Task 5)
  - `POST /api/family/regenerate-code/` â†’ `@action detail=False, methods=['post']` â€” admin only, generate new code, reset expires_at
  - `DELETE /api/family/members/<user_id>/` â†’ `@action detail=False, methods=['delete']` â€” admin only, remove member; cannot remove self (use leave instead)

### Task 5: Update Gasto views to filter by family
**File:** `backend/api/views.py`
- `GET /api/gastos/`: if user has family, return gastos where `family=user.membership.family` OR `user=request.user` (own gastos without family); if no family, return only own gastos
- `POST /api/gastos/`: set `family` from user's membership if present, set `user=request.user`
- `PUT/PATCH /api/gastos/<id>/`: permission â€” allow if user is creator OR (user is admin of same family AND gasto was created after member joined family)
- `DELETE /api/gastos/<id>/`: same permission as PUT
- Add custom permission class `GastoPermission` in `permissions.py`
